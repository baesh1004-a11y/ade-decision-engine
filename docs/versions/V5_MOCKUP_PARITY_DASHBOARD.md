# V5 Mockup Parity Dashboard

## 목적

기존 V4 추천 워크벤치는 기능은 통합했지만, 설계 시안과 실제 Streamlit 화면의 구조와 정보 밀도가 크게 달랐다. V5는 PC 대시보드 시안과 최대한 같은 작업 흐름과 화면 구성을 구현한다.

## 백업

변경 전 V4는 다음 브랜치에 보존한다.

```bash
git switch backup/v4-before-mockup-parity
```

기준 커밋: `f5c8e878af0b24629a554175737d502397b1a88f`

## 구현 구조

1. 상단 KPI 6개
   - 오늘 추천 종목
   - 평균 유사도
   - 매수 검토
   - 관찰 종목
   - 주문 대기
   - 최근 실행

2. 중앙 4단계 PC 워크플로
   - 추천 생성
   - 추천 근거 비교
   - 추천 검증
   - 주문 관리

3. 하단 후속 정보
   - 과거 급등 사례
   - 추천 근거
   - 보유 및 성과 관리

## 종목명 표시

추천 목록은 종목명, 종목코드, 최종 유사도, 주봉 유사도, STO 유사도를 함께 표시한다. `daily_recommendations.name`이 비어 있으면 `payload_json.name`을 사용하고, 둘 다 없을 때만 종목코드를 표시한다.

## 설계 원칙

- 추천 점수는 추천 생성 결과를 그대로 사용한다.
- 추천 근거 비교는 순위나 판정을 다시 계산하지 않는다.
- 추천 검증은 시장, 업종, 위험 체크리스트로 표시한다.
- 주문은 기존 주문관리 페이지에서 사용자 승인 후 실행한다.
- Streamlit 기본 컴포넌트를 사용하므로 이미지 시안과 완전한 픽셀 단위 동일성은 보장하지 않지만, 화면 분할, 정보 순서, 카드 구성과 작업 흐름은 동일하게 맞춘다.
